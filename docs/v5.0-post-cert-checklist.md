# v5.0 - Post-cert shipping checklist

Reference when the Apple Developer Program enrollment email lands. Everything below assumes the code-side migration from PR #144 is merged or still on the `feat/apple-dev-migration` branch.

Total ballpark time from cert arrival to `v5.0.0` published: **~90 min focused** (most of it is waiting for builds and portal propagation).

---

## Phase A - Cert + Xcode setup (10 min)

- [ ] Apple Developer Program enrollment email arrived + account status is "Active"
- [ ] Xcode -> Settings -> Accounts: sign in with the Apple ID used for enrollment
- [ ] Same screen -> Manage Certificates -> + -> add both:
  - [ ] `Apple Development`
  - [ ] `Developer ID Application`
- [ ] Note the Team ID (10 chars, e.g. `ABCDE12345`):
  ```bash
  security find-identity -v -p codesigning | grep "Developer ID Application"
  # The ID in parens is your Team ID
  ```

---

## Phase B - Developer Portal (10 min)

Sign in to https://developer.apple.com/account -> Certificates, Identifiers & Profiles.

- [ ] **Identifiers -> App Groups -> `+`**
  - [ ] Description: `TokenEater App Group`
  - [ ] Identifier: `group.com.tokeneater` (must match exactly)
- [ ] **Identifiers -> App IDs** - verify both exist, create if not:
  - [ ] `com.tokeneater.app` (main app)
  - [ ] `com.tokeneater.app.widget` (widget extension)
- [ ] Edit each App ID -> Capabilities -> enable **App Groups** -> tick `group.com.tokeneater` -> Save
- [ ] Double-check the App Group now shows "Enabled (2 App IDs)" in the portal listing

---

## Phase C - GitHub repo secrets (15 min)

### Export the Developer ID cert as .p12

- [ ] Open **Keychain Access**
- [ ] Search for `Developer ID Application: <your name>`
- [ ] Right-click -> **Export** -> save as `tokeneater-devid.p12` -> set a strong password (you'll paste it in a secret)
- [ ] Base64-encode for GitHub:
  ```bash
  base64 -i ~/Downloads/tokeneater-devid.p12 | pbcopy
  ```

### Generate an app-specific password for notarytool

- [ ] Go to https://appleid.apple.com -> **Sign-In and Security** -> **App-Specific Passwords** -> `+`
- [ ] Label: `TokenEater Notarization`
- [ ] Copy the generated `xxxx-xxxx-xxxx-xxxx` password

### Add all secrets

At https://github.com/AThevon/TokenEater/settings/secrets/actions, click **New repository secret** for each:

- [ ] `APPLE_CERT_P12_BASE64` - paste clipboard from the base64 step
- [ ] `APPLE_CERT_PASSWORD` - the password used when exporting the .p12
- [ ] `APPLE_ID` - your Apple Dev account email
- [ ] `APPLE_APP_PASSWORD` - the app-specific password from above
- [ ] `APPLE_TEAM_ID` - 10-char Team ID from Phase A

### Sanity-check existing secrets (should already be there)

- [ ] `SPARKLE_PRIVATE_KEY` - EdDSA key used to sign appcast entries (unchanged)
- [ ] `HOMEBREW_TAP_TOKEN` - token for the brew cask bump step (unchanged)

---

## Phase D - Local code sync (10 min)

From the `feat/apple-dev-migration` branch:

- [ ] Reinject the App Group entitlement:
  ```bash
  ./scripts/enable-app-groups.sh
  ```
  Verifies + edits `TokenEaterApp.entitlements` + `TokenEaterWidget.entitlements`. Also drops the widget `temporary-exception`.

- [ ] Update `project.yml` with the real signing config. Replace:
  ```yaml
  settings:
    base:
      CODE_SIGN_STYLE: Automatic
  ```
  with:
  ```yaml
  settings:
    base:
      CODE_SIGN_STYLE: Manual
      DEVELOPMENT_TEAM: "YOUR_TEAM_ID"  # from Phase A
    configs:
      Debug:
        base:
          CODE_SIGN_IDENTITY: "Apple Development"
      Release:
        base:
          CODE_SIGN_IDENTITY: "Developer ID Application"
  ```

- [ ] Regenerate + verify the project builds:
  ```bash
  xcodegen generate
  xcodebuild -project TokenEater.xcodeproj -scheme TokenEaterApp -configuration Release \
    -derivedDataPath build build 2>&1 | tail -3
  ```
  Expect `** BUILD SUCCEEDED **`.

- [ ] Commit the post-cert changes:
  ```bash
  git add project.yml TokenEaterApp/TokenEaterApp.entitlements TokenEaterWidget/TokenEaterWidget.entitlements
  git commit -m "build: activate Developer ID signing + App Group entitlement"
  git push
  ```

## Phase E - Validation (30 min)

### Local smoke test (ad-hoc - should still work)

- [ ] Full nuke + reinstall (see `CLAUDE.md` mega-nuke block)
- [ ] App launches without admin prompt (helper is gone in v5.0)
- [ ] Menu bar shows data
- [ ] Popover opens
- [ ] Dashboard renders all metrics

### CI test release (end-to-end signing + notarization)

- [ ] Tag a release candidate:
  ```bash
  git tag v5.0.0-rc.1 HEAD
  git push origin v5.0.0-rc.1
  ```
- [ ] Monitor https://github.com/AThevon/TokenEater/actions - expect the Release workflow to:
  - Import the cert
  - Build with Manual signing + Hardened Runtime
  - Sign the DMG
  - Notarize via notarytool (can take 3-10 min)
  - Staple the ticket
- [ ] If the notarize step fails, check the notarytool log URL it prints
- [ ] If everything green, download the DMG from the rc.1 release page

### Fresh-user-account install test (optional but recommended)

- [ ] Create a new macOS user account (System Settings -> Users & Groups -> +)
- [ ] Log into that account, download the rc.1 DMG
- [ ] Open the DMG + drag to Applications + double-click TokenEater
- [ ] Expect: **opens directly, no Gatekeeper prompt, no "Open Anyway" needed**
- [ ] If Gatekeeper still blocks, stapling failed - check `xcrun stapler validate TokenEater.dmg`

### Upgrade path test (critical)

- [ ] Install v4.12.0 on a clean account, use it a bit (complete onboarding, pin a couple metrics, change theme)
- [ ] Install v5.0.0-rc.1 on top
- [ ] Verify:
  - Prefs carried over (pinnedMetrics, theme, onboarding-complete flag preserved)
  - No onboarding re-shown
  - Legacy helper is gone: `launchctl list | grep tokeneater` returns nothing
  - `~/Library/LaunchAgents/com.tokeneater.helper.plist` gone
  - Menu bar works, popover opens, no "Claude Safe Storage" prompt

## Phase F - Ship v5.0.0 (15 min)

- [ ] Bump version in `project.yml`:
  ```yaml
  MARKETING_VERSION: "5.0.0"
  ```
- [ ] Run `xcodegen generate` to apply
- [ ] Commit:
  ```bash
  git add project.yml
  git commit -m "bump: v5.0.0"
  git push
  ```
- [ ] Merge PR #144 on GitHub (squash)
- [ ] Tag + push:
  ```bash
  git fetch origin main
  git tag v5.0.0 origin/main
  git push origin v5.0.0
  ```
- [ ] Wait for release workflow (~5 min)
- [ ] Edit the GitHub release with curated notes (use the PR body as the template - same approach as v4.12.0)
- [ ] Verify:
  - [ ] DMG downloadable from https://github.com/AThevon/TokenEater/releases/latest
  - [ ] `docs/appcast.xml` has the v5.0.0 entry with a valid `edSignature`
  - [ ] `AThevon/homebrew-tokeneater` `Casks/tokeneater.rb` bumped to 5.0.0

---

## Phase G - Post-release monitoring (1-2 weeks)

- [ ] Watch GitHub Issues for regressions (especially: missing onboarding, keychain prompts, launch crashes)
- [ ] Watch Reddit / Twitter for user reports
- [ ] Keep an eye on download numbers in GitHub release stats - if a spike in redownloads suggests brew cask breakage

If serious regression: `git revert` the merge on `main` -> tag a v5.0.1 with the revert -> release.
